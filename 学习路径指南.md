编辑 `config.json`，启用 Agent 模式：

```json
  "agent_max_context_turns": 20,

}
```

然后重启 `python app.py`，和 Agent 聊天试试！

**1.3 理解项目结构**

| 目录 | 作用 | 重点程度 |
|------|------|---------|
| `app.py` | 入口，启动所有通道 | ⭐⭐⭐⭐⭐ |
| `config.py` | 配置加载和管理 | ⭐⭐⭐⭐ |

**1.4 核心概念速查**

```
Session（会话）
├── 每个用户/群聊 = 一个 Session
├── session_id = 用户ID（私聊）或 群ID（群聊）
└── 每个 Session 独立的消息队列 + 并发控制

Channel（通道）
├── 每个接入平台 = 一个 Channel
├── Channel 有两类：
│   ├── ChatChannel（通用）→ 微信/飞书/钉钉/QQ/Web
│   └── 特殊通道 → 公众号/企业微信/终端
└── 每个 Channel 独立线程运行

Plugin（插件）
├── 事件驱动的扩展点
├── 按优先级顺序执行
└── 可以拦截、修改、中断消息处理

Agent（智能体）
├── 思考-行动循环（ReAct 模式）
├── 调用工具完成复杂任务
└── 有记忆，能跨会话学习
```

## 📖 Day 2-3：吃透插件系统（最重要！）

### 目标：理解事件驱动架构，写出自己的插件

**2.1 为什么插件系统是 CowAgent 的灵魂？**

```
传统写法（if-else 地狱）：
if 消息 == "Hello": reply("Hello")
elif 消息 == "画图": 调用画图()
elif 消息 == "搜素": 调用搜索()
elif 消息.startswith("#"): 处理命令()
...

CowAgent 写法（事件驱动）：
消息到达 → emit_event(ON_HANDLE_CONTEXT)
              ↓
         插件A (关键词回复)  → BREAK_PASS
         插件B (角色扮演)    → CONTINUE
         插件C (Agent)      → 调模型处理
```

**2.2 四种事件，按消息生命周期排序**

```
消息到达 ──→ ON_RECEIVE_MESSAGE ──→ ON_HANDLE_CONTEXT ──→ ON_DECORATE_REPLY ──→ ON_SEND_REPLY
  │              │                        │                      │                    │
  │           过滤/修改               生成回复               格式化回复             最终发送
  │           session_id                                    前缀后缀              拦截点
  │                                 拦截/修改                                   
  │                                 BREAK/BREAK_PASS
```

| 事件 | 时机 | 常用场景 |
|------|------|---------|
| `ON_RECEIVE_MESSAGE` | 收到消息最早 | 过滤消息、修改 session_id |
| `ON_HANDLE_CONTEXT` | **最常用** | 关键词回复、角色扮演、Agent 拦截 |
| `ON_DECORATE_REPLY` | 回复生成后 | 修改格式、添加前后缀、语音转文字 |
| `ON_SEND_REPLY` | 发送前最后 | 日志记录、最终拦截 |

**2.3 三种动作，决定消息怎么处理**

```python
# CONTINUE：我只是路过，你们继续（插件通常不用这个）
e_context.action = EventAction.CONTINUE

# BREAK：我来改消息内容，然后让模型处理（修改后让AI处理）
e_context["context"].content = "新内容"
e_context.action = EventAction.BREAK

# BREAK_PASS：我已经搞定了，模型不用管了（插件自己回答）
e_context["reply"] = Reply(ReplyType.TEXT, "我自己回答了")
e_context.action = EventAction.BREAK_PASS
```

**2.4 写一个插件（照着 hello 插件改）**

```python
# plugins/my_plugin/__init__.py（必须是这个结构）
# plugins/my_plugin/my_plugin.py

from plugins import *
from bridge.reply import Reply, ReplyType
from plugins.event import EventContext, Event, EventAction

@plugins.register(
    name="MyPlugin",           # 插件名字
    desire_priority=90,       # 优先级（越大越先执行）
    desc="我的第一个插件",      # 描述
    author="me",               # 作者
    version="1.0"             # 版本
)
class MyPlugin(Plugin):
    def __init__(self):
        super().__init__()
        # 在这里注册事件处理函数
        # 最常用的是 ON_HANDLE_CONTEXT
        self.handlers[Event.ON_HANDLE_CONTEXT] = self.on_handle_context
    
    def on_handle_context(self, e_context: EventContext):
        """处理消息"""
        content = e_context["context"].content
        
        # === 方式1：直接回复，不调模型 ===
        if "你好" in content:
            reply = Reply(ReplyType.TEXT, "你好呀！")
            e_context["reply"] = reply
            e_context.action = EventAction.BREAK_PASS  # 重要！
            return
        
        # === 方式2：修改消息，让模型处理 ===
        if "帮我" in content:
            e_context["context"].content = content + "，请用友好的语气回答"
            e_context.action = EventAction.BREAK
            return
        
        # === 方式3：改变消息类型 ===
        if content.startswith("画:"):
            e_context["context"].type = ContextType.IMAGE_CREATE
            e_context["context"].content = content[2:]  # 去掉"画:"前缀
            e_context.action = EventAction.CONTINUE  # 继续传给下一个插件
            return
    
    def get_help_text(self, **kwargs):
        """用户输入 #help 时显示的帮助信息"""
        return "这个插件的功能：\n- 包含'你好'会回复问候"
```

## 📖 Day 4：理解 Agent 模式

### 目标：理解"思考-行动"循环，写出自己的工具

**4.1 普通模式 vs Agent 模式**

```
普通模式（直接调模型）：
用户消息 → 模型 → 回复

Agent 模式（思考后行动）：
用户消息 → 思考 → 决定调用工具 → 执行工具 → 结果 → 继续思考 → ...
                                              ↓
                                         重复直到完成
```

**4.2 Agent 怎么工作？**

```
┌────────────────────────────────────────────────────┐
│  Agent.run_stream()  主循环                        │
│                                                     │
│  1. 构建 System Prompt                              │
│     → 工具定义（给LLM看的工具有哪些）                │
│     → Skills 内容（给LLM看的技能说明）               │
│     → 记忆上下文（相关记忆片段）                    │
│     → 运行时信息（当前时间等）                      │
│                                                     │
│  2. 调用 LLM                                        │
│     → LLM 返回文本？ → 完成，返回给用户              │
│     → LLM 返回工具调用？ → 执行工具 → 回到第2步      │
│                                                     │
│  3. 重复直到 LLM 返回文本 或 达到最大步数            │
└────────────────────────────────────────────────────┘
```

**4.3 写一个工具（参考 bash_tool）**

```python
# agent/tools/my_tool/my_tool.py

from agent.tools.base_tool import BaseTool, ToolResult
from common.log import logger

class MyTool(BaseTool):
    """我的自定义工具"""
    
    name = "my_tool"                    # 工具名（唯一标识）
    description = "做某件事的工具"       # 给LLM看的描述（决定什么时候用）
    
    # 参数定义（JSON Schema 格式）
    params = {
        "type": "object",
        "properties": {
            "query": {
                "type": "string",
                "description": "查询内容"
            }
        },
        "required": ["query"]
    }
    
    def execute(self, params: dict) -> ToolResult:
        """具体执行逻辑"""
        query = params.get("query")
        
        try:
            # 你的逻辑...
            result = f"查询结果: {query}"
            return ToolResult.success(result)
        except Exception as e:
            logger.error(f"MyTool 执行失败: {e}")
            return ToolResult.fail(str(e))
```

**4.4 工具vs插件，区别是什么？**

| 维度 | 插件 | 工具 |
|------|------|------|
| **触发方式** | 事件驱动（消息到达就触发） | Agent 调用（LLM 决定用哪个） |
| **执行时机** | 消息处理流水线中 | Agent 思考循环中 |
| **用途** | 拦截消息、过滤内容、添加功能 | 执行具体任务（搜索、画图、代码执行...） |
| **是否需要 LLM 决策** | 否（插件自己决定怎么响应） | 是（LLM 决定是否调用） |

**4.5 工具分类（CowAgent 自带的）**

| 工具 | 文件 | 作用 |
|------|------|------|
| bash | `tools/bash/bash.py` | 执行命令行 |

## 📖 Day 5：理解记忆系统

### 目标：理解 Agent 怎么记住跨会话的信息

**5.1 记忆架构**

```
┌─────────────────────────────────────────────────────────────┐
│                     MemoryManager                            │
│                    （记忆管理器）                             │
│                                                             │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐    │
│  │  MEMORY.md  │    │ daily/*.md  │    │ users/*/*.md│    │
│  │ (核心记忆)   │    │ (日级记忆)   │    │(用户级记忆) │    │
│  └──────┬──────┘    └──────┬──────┘    └──────┬──────┘    │
│         │                  │                  │            │
│         └──────────────────┼──────────────────┘            │
│                            ▼                                │
│                 ┌──────────────────┐                        │
│                 │  TextChunker    │  ← 文本分块             │
│                 └────────┬─────────┘                        │
│                          ▼                                 │
│                 ┌──────────────────┐                        │
│                 │ EmbeddingProvider│ ← 向量嵌入              │
│                 └────────┬─────────┘                        │
│                          ▼                                 │
│                 ┌──────────────────┐                        │
│                 │  MemoryStorage    │ ← SQLite + FTS5       │
│                 └──────────────────┘                        │
│                            │                                │
│                            ▼                                │
│                 ┌──────────────────┐                        │
│                 │   混合检索       │ ← 向量+关键词+时间衰减  │
│                 └──────────────────┘                        │
└─────────────────────────────────────────────────────────────┘
```

**5.2 记忆是怎么积累的？**

```
对话进行中...
        ↓
  Agent 回复用户
        ↓
  Agent.memory_manager.flush_memory()
        ↓
  把最近的对话摘要写入 memory/daily/2024-01-29.md
        ↓
  MemoryManager._sync_file() 扫描文件
        ↓
  分块 → 嵌入 → 存入 SQLite
        ↓
  下次对话时，搜索相关记忆 → 注入 System Prompt
```

**5.3 检索公式**

```
最终得分 = (0.7 × 向量相似度 + 0.3 × 关键词匹配) × 时间衰减

时间衰减（对日级记忆）：
- 今天的记忆：× 1.0
- 7天前：× 0.85
- 30天前：× 0.5（半衰期）
- 90天前：× 0.12
- MEMORY.md：× 1.0（永不衰减）
```
